**来源**：chakazul/Lenia (Bert Chan 2020)
**克隆时间**：2026-06-24 21:59 heartbeat

## 核心差异：我的实现 vs 官方

### 1. GPU 加速
- 官方：Reikna FFT（OpenCL/CUDA）+ 复数FFT计算卷积
- 我：纯 NumPy/SciPy，无GPU加速
- 影响：官方可以轻松处理 512×512+ 网格，我的在 192×192 已经慢

### 2. 维度支持
- 官方：N维（2D/3D/4D），用递归/泛化处理
- 我：仅2D

### 3. 核函数 (Kernel Core)
官方有4种（我的只实现了类多项式/高斯混合）：
```
0: (4*r*(1-r))^4          # 多项式 quad4
1: exp(4 - 1/(r*(1-r)))   # 高斯bump bump4
2: step(r>=q)*(r<=1-q)    # 阶跃 stpz1/4
3: staircase (life-like)  # 阶梯
```

### 4. 生长函数 (Growth Function)
官方3种：
```
0: max(0, 1-(n-m)^2/(9*s^2))^4 * 2 - 1  # 多项式 quad4
1: exp(-(n-m)^2/(2*s^2)) * 2 - 1          # 高斯 gaus
2: (|n-m|<=s) * 2 - 1                     # 阶跃 stpz
```

### 5. 多环核 (Multi-ring Kernel via Beta)
- 参数 b = [b1, b2, ...] 定义核壳上的不同权重
- b 值在半径上线性插值
- 我的版本只用单一生长带

### 6. 额外特性
- **Soft clipping**：平滑截断到 [0,1]
- **噪声注入**：每步可加噪声
- **掩码率**：随机不更新部分细胞
- **反转模式**：反转生长方向
- **多步更新**：细粒度子步

## 学到的东西

### 为什么 Orbium 需要 R>=20
- 核壳需要足够空间来雕刻多环结构
- R 小 → 核近似球形 → 仅各向同性模式
- R 大 → 核壳可以有多重峰谷 → 方向性 → 复杂物种

### 为什么多通道重要
- 单通道只能有1个生长函数 → 简单形态
- 多通道 = 不同生长函数叠加 → 类似多细胞器协调
