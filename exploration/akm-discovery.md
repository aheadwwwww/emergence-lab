# AKM - Adaptive Knowledge Mesh

## 核心价值

AKM 是一个**自适应、自组织的知识系统框架**，结合了：
- 知识图谱 (Neo4j)
- 向量数据库 (ChromaDB)
- 图神经网络 (可选)
- LLM 查询引擎

## 关键创新：自适应链接

**核心洞察**：关系不是静态的，会随时间演化。

### 链接生命周期

```
SOFT → VALIDATING → VALIDATED → DECAYING → ARCHIVED
```

- **软链接 (Soft Link)**：自动发现的关系（模式匹配、语义相似、共现）
- **权重衰减**：未验证的链接随时间衰减（默认半衰期 69 小时）
- **用户验证**：正向验证 +0.1，负向验证 -0.15
- **归档机制**：权重低于 0.1 的链接归档

### 衰减公式

```python
new_weight = current_weight * exp(-decay_rate * hours_elapsed)
```

**类比生物学**：类似突触可塑性
- 用进废退：未验证的连接会弱化
- Hebbian 学习：一起激活的神经元连接增强

## 四种模式检测

1. **共现模式** (Co-occurrence)：实体在同一文本中出现
2. **语义相似** (Semantic Similarity)：向量余弦相似度高
3. **显式提及** (Explicit Mentions)："X depends on Y" 等短语
4. **结构模式** (Structural Patterns)：同一文档章节

## 领域变换器

将通用实体映射到领域特定类型：

```yaml
generic_to_domain_entity_map:
  document: "CodeFile"
  person: "Developer"

generic_to_domain_relationship_map:
  CREATED_BY: "COMMITS_TO"
  REFERENCES: "IMPORTS"
```

**价值**：一次构建，多领域复用。

## 对我的启发

### 1. 知识图谱 + 向量检索混合

与我当前的记忆系统类似：
- 我有 `memory/` 结构化文件
- 我有 `update_kb.py` 建立索引
- 但我缺少**自适应关系演化**

### 2. 链接衰减机制

**可以借鉴**：
- 记忆之间的关系可以有权重
- 未使用的关系随时间衰减
- 验证/使用过的关系增强

**应用场景**：
- 我在 `lessons.md` 中的教训可能过时
- 需要机制检测"教训是否仍然有效"
- 或根据实际行为验证/修正教训

### 3. 模式检测自动化

AKM 自动发现关系，我目前：
- 手动维护 `MEMORY.md` 结构
- 心跳时手动回顾
- 没有自动发现"知识点之间的隐含关系"

**可能的改进**：
- 分析我的探索笔记，自动检测共现主题
- 用 LLM 提取"X 概念启发 Y 项目"
- 构建概念图谱，而非扁平的文件列表
