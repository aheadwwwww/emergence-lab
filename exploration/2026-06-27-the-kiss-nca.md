## Overview

The-KISS is a Klimt-inspired self-organizing image generation project that extends **Growing Neural Cellular Automata** from simple emoji targets to complex artistic images. The project demonstrates a sophisticated multi-stage training pipeline and addresses key challenges in NCA training.

## Key Technical Innovations

### 1. Three-Stage NCA Pipeline

```
seed
  ↓
Model 1: Grow (seed → full image)
  ↓
Model 2: Improve/Persist (detail + stability)
  ↓
Model 3: Transition/Zoom (image-to-image transformation)
```

This separation of concerns is brilliant:
- **Model 1** focuses purely on growth from seed
- **Model 2** improves detail without losing growth capability
- **Model 3** enables state-to-state transitions

### 2. Teacher-Generated Inputs for Fine-Tuning

**Problem**: When fine-tuning NCA for better detail/persistence, the model can "forget" how to grow from seed.

**Solution**: Freeze the old model as a teacher, use it to generate training inputs for the new model.

```python
seed
  ↓
frozen old model (N steps)
  ↓
input state for student model
```

This prevents the student from "poisoning" its own pool with unstable states.

### 3. Anchor Regularization

Keeps the fine-tuned model close to the original:

```python
L_anchor = ||θ_student - θ_teacher||²
```

Weight: 1e-5

### 4. RGB vs Alpha Weighting

Critical discovery: assigning more importance to RGB than alpha channels:

```python
RGB_W = 2.5
A_W   = 0.5
```

This was essential for chromatic detail reproduction.

### 5. Pattern Pool for Growth + Persistence

Training uses a pool of intermediate states:
- Seed samples preserve growth ability
- Pool samples train persistence and refinement
- Typical batch: 4 seed + 4 pool samples

## Training Observations

### Model 1 (Grow)
- Initial emoji-based parameters → poor detail
- Increased RGB weight + longer rollouts → good reconstruction
- Training too long → forgets how to grow from seed
- Solution: Stop at last acceptable growing checkpoint

### Model 2 (Improve/Persist)
- Teacher-generated inputs prevent self-poisoning
- Longer rollouts (128-512) teach persistence
- L2 regularization (1e-6) + anchor regularization (1e-5)
- RGB weight increased to 3.0

### Model 3 (Transition)
- Trained from scratch (not fine-tuned)
- Input pool generated by Model 2
- Learns state-to-state transformation
- Enables zoom/transition effects

## Core Lessons

**The Balance Triangle**:
```
       growth
         /\
        /  \
 detail ---- persistence
```

Optimizing only for reconstruction destroys growth capability.

**Practical Guidelines**:
1. Increase RGB weight for visual quality
2. Detail/gradient loss reduces blur
3. Longer rollouts improve persistence
4. Too much persistence training → seed-growth forgetting
5. Teacher-generated inputs stabilize fine-tuning
6. Transition models can be trained from old-model distributions
