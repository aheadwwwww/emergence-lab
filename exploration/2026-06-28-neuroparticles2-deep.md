# neuroparticles2 深度分析

## 项目概览
- **作者**: xcontcom
- **Stars**: 69
- **语言**: JavaScript (vanilla, browser-based)
- **核心**: 粒子+细胞自动机+遗传算法三层融合的人工生命

## 架构分析

### 三层融合机制

```
基因型(512bit) → CA规则 → 环境感知(13x13→3x3) → 移动方向 → 适应度 → 进化选择
     ↑                                                              ↓
     └──────────────── 交叉+变异（每1000步）──────────────────────┘
```

### 1. 粒子层
- 200个粒子在256×256网格上
- 每个粒子携带512位基因型（二进制串）
- 位置不重叠（初始化用集合去重）
- 移动后周期性边界环绕

### 2. CA层
- 以粒子为中心提取13×13区块
- 递归CA步进：13→11→9→7→5→3（每次CA后缩2像素）
- CA规则由粒子基因型决定（512位 → 18位规则 → 512条规则表）
- 最终3×3输出 → 加权投票决定移动方向
- 使用sign而非round：防止小数舍入丢失方向信息

### 3. GA层
- 每1000步进化一次
- 适应度 = 邻居数量（+1/邻居） - 碰撞惩罚（-3/重叠）
- 选择：按适应度排序，前50%存活
- 繁殖：每4个位置由2个父代交叉产生2个子代
- 变异：mutation%概率，每次翻转1-16个位
- 适应度累积而非重置（跨代累积）

### 核心算法

**基因型→CA规则映射**：
- 512位基因型直接用作CA查表
- 用9位邻居模式（512种）索引基因型
- 支持所有可能的2D CA规则空间

**递归感知**：
- evaluate(field, rule, x, y) 递归收缩
- 类似CNN的多层感受野，但用CA替代卷积
- 13→3收缩让粒子"看到"周围环境

**移动决策**：
- 3×3 CA输出 → 9方向加权投票
- sign(dx), sign(dy) 量化到9方向
- total==0时静止

## 设计模式可借鉴

### 1. 基因型即规则
- 不预设行为，一切从基因型+环境涌现
- 类比：我们的Neural Lenia可以用神经网络权重作为"基因型"

### 2. 递归感知收缩
- 类似CNN但不使用训练权重
- 可借鉴：Lenia中让核函数收缩感知范围

### 3. 累积适应度
- 不每代清零，跨代累积
- 让长期表现好的个体持续受青睐

### 4. 纯前端可视化
- Canvas渲染，实时观察涌现
- 可借鉴：为emergence-lab做浏览器端可视化

## 与Lenia的对比

| 维度 | neuroparticles2 | Lenia |
|------|----------------|-------|
| 空间 | 离散256×256 | 连续场 |
| 主体 | 200个粒子 | 每个细胞 |
| 规则 | 512bit基因型 | 核函数 |
| 进化 | 每1000步GA | 无（Neural Lenia可引入） |
| 时间 | 离散步 | 连续dt |
| 感知 | 递归CA收缩 | 固定R半径卷积 |

## 启示

1. **进化+涌现是powerful组合**：Lenia也可以引入进化（核函数参数作为基因型）
2. **简单感知机制产生复杂行为**：递归CA收缩比固定半径卷积更丰富
3. **浏览器可视化降低门槛**：emergence-lab可以考虑Web前端
4. **适应度函数设计是关键**：邻居奖励+碰撞惩罚产生了群集行为
