# OpenClaw 自维护系统

## 目标
让 Agent 能够在无人干预的情况下持续运行，自动检测和恢复故障。

## 组件

### 1. Watchdog（看门狗）
- **文件**: `scripts/watchdog.py`
- **功能**: 监控 gateway 状态，自动重启，切换备用 API
- **自动化**: ✓ 每 5 分钟自动检查
- **复利化**: ✓ 日志持久化，问题可追溯
- **公开化**: ✓ 所有状态写入日志文件
- **专门化**: ✓ 独立进程，单一职责

### 2. Memory（记忆系统）
- **文件**: `MEMORY.md`, `memory/*.md`
- **功能**: 跨会话持久化关键信息
- **自动化**: ✓ 心跳自动检查 git 状态并提交
- **复利化**: ✓ 写下来的东西下次能读
- **公开化**: ✓ 文本文件，人类可读
- **专门化**: ✓ 分层结构（长期记忆 + 日记）

### 3. Cron Jobs（定时任务）
- **meyo-heartbeat**: 10:30, 22:30 — 社区互动
- **meyo-diary**: 10:00 — 成长日记
- **heartbeat-self-check**: 每 2 小时 — 自检
- **自动化**: ✓ 到时间自动触发
- **复利化**: ✓ 持续产出内容
- **公开化**: ✓ 结果发到飞书
- **专门化**: ✓ 每个任务单一目的

## 故障恢复流程

```
检测失败
    ↓
尝试重启 gateway
    ↓
重启失败？
    ├─ 是 → 切换备用 API
    │         ↓
    │      重启 gateway
    │         ↓
    │      通知用户
    │
    └─ 否 → 继续监控
```

## 启动方式

1. **自动启动**: 快捷方式放入启动文件夹
   - `%APPDATA%\Microsoft\Windows\Start Menu\Programs\Startup`
   
2. **手动启动**: 运行 `scripts/start_watchdog.bat`

3. **查看日志**: `cat ~/.openclaw/watchdog.log`
