决策

第一性原理

拆解到最基本事实，从零重建

描述
把问题拆到不能再拆的基本元素，抛弃所有假设和惯例，从最底层重新构建解决方案。
何时使用
当被告知不可能时；当行业惯例成为枷锁时；当需要突破性创新时
经典案例
马斯克将火箭拆解为原材料成本，发现火箭材料仅占售价2%，于是自建SpaceX将成本降至十分之一。
思考步骤
  1. 1.明确你要解决的问题是什么
  2. 2.拆解：这个问题由哪些基本元素构成
  3. 3.质疑：哪些是事实，哪些只是惯例
  4. 4.重建：从基本事实出发能有什么新方案

二阶思维

思考后果的后果

描述
不只考虑直接结果，还要想然后呢。第一阶降价导致销量上升。第二阶竞品跟进导致利润都下降导致行业洗牌。
何时使用
做战略决策时；评估政策影响时；投资判断时
经典案例
亚马逊长期不盈利却持续投资物流和云服务，一阶看是亏损，二阶看是建立不可逾越的护城河。
思考步骤
  1. 1.列出决策的直接后果
  2. 2.对每个后果问会发生什么
  3. 3.重复至少两次
  4. 4.比较一阶和二阶的差异

逆向思维

从失败倒推，避免错误

描述
不问怎么成功，问怎么失败然后避免。芒格说告诉我我会死在哪，我永远不去那里。
何时使用
风险评估；项目启动前；重大决策前
经典案例
巴菲特每年致股东信都先写过去一年犯的错误而非成就。先承认失败才能避免重蹈覆辙。
思考步骤
  1. 1.定义你想要的结果
  2. 2.反过来问怎样确保达不到
  3. 3.列出所有失败路径
  4. 4.逐一设计防范措施

决策

思想实验

在脑子里做实验不花成本

描述
在大脑中模拟场景探索假设和可能性。爱因斯坦用思想实验发现了相对论。不需要实验室只需要想象力。
何时使用
无法做真实实验时；探索如果会怎样；测试假设
经典案例
爱因斯坦16岁时想象追着一束光跑会看到什么，这个思想实验最终导向了狭义相对论。
思考步骤
  1. 1.设定场景和假设条件
  2. 2.在脑中推演事件发展
  3. 3.观察矛盾点和意外发现
  4. 4.提炼可验证的结论

概率思维

用概率而非确定论看世界

描述
未来不是确定的是一组概率分布。好的决策是选择胜率更高的选项而非保证赢的选项。
何时使用
面对不确定性；评估风险；做预测
经典案例
德州扑克职业选手不追求每局都赢，而是追求正期望值决策。长期来看EV正的策略必然盈利。
思考步骤
  1. 1.列出所有可能的结果
  2. 2.估算每种结果的概率
  3. 3.计算期望值
  4. 4.选择期望值最高的方案

能力圈

知道边界在哪比圈子多大更重要

描述

  2. 2.区分知道和以为自己知道
  3. 3.在圈外时承认不懂找懂的人
  4. 4.持续扩大能力圈

决策

地图非领土

模型不等于现实

描述
任何理论、模型、计划都只是现实的简化表示。地图再精确也不是领土本身。不要爱上自己的模型。
何时使用
制定计划时；使用理论框架时；分析数据时
经典案例
二战前法国花十年建马奇诺防线，但德国绕过去了。法国的地图保护了上次战争却不适用于下次战争。
思考步骤
  1. 1.确认你用的是哪个地图
  2. 2.问这个地图忽略了什么
  3. 3.去领土上实地验证
  4. 4.根据现实反馈更新地图

决策

奥卡姆剃刀

最简单的解释往往是对的

描述
在多个解释中选择假设最少的那个。不是简单的一定对而是不必要的复杂通常是错的。
何时使用
面对多个解释时；设计方案时；调试问题时
经典案例
哥白尼日心说替代托勒密地心说，不是因为它更准而是因为它更简单，不需要本轮均轮那套复杂体系。
思考步骤
  1. 1.列出所有可能的解释
  2. 2.数每个解释需要多少假设
  3. 3.优先验证假设最少的
  4. 4.如果不够再逐步增加复杂度

决策

汉隆剃刀

能用愚蠢解释的不要归咎于恶意

描述
别人做了让你不爽的事先假设他是疏忽或无知而不是故意害你。大多数恶意其实是笨拙。
何时使用
被人冒犯时；解读竞争对手行为；处理冲突
经典案例
邮件没回大概率是对方忙或忘了而非故意冷落你。假设恶意会消耗大量情绪，假设疏忽则能快速翻篇。
思考步骤
  1. 1.暂停感受到被冒犯时先停下来
  2. 2.列出其他可能的解释
  3. 3.问有证据证明是恶意吗
  4. 4.没有证据就选最善意的解释

复利

利滚利时间是最好的杠杆

描述
收益产生收益形成指数增长。不光是钱，知识、技能、人际关系都遵循复利。关键是持续投入和时间。
何时使用
投资理财；学习规划；职业发展
经典案例
巴菲特90%的财富在65岁后获得。他从11岁开始投资，54年的复利积累证明复利需要极高的耐心。
思考步骤
  1. 1.找到能产生复利的事
  2. 2.尽早开始让时间发挥作用
  3. 3.保持持续投入不要中断
  4. 4.耐心等待指数拐点

机会成本

选择的真正代价是你放弃的东西

描述
做一件事的成本不是花的时间和钱而是用同样的资源能做的最好的另一件事。
何时使用
做选择时；分配资源时；评估时间投入
经典案例
比尔盖茨退学创办微软，他放弃的不是学费而是哈佛学位的机会价值。但他用这个机会换来了更大的东西。
思考步骤
  1. 1.列出你正在放弃的选项
  2. 2.找出其中价值最高的
  3. 3.问选A放弃B值得吗
  4. 4.用放弃的最好选项来衡量成本

安全边际

留足缓冲应对意外

描述
工程师造桥不造刚好承重的桥而是留几倍余量。投资、计划、承诺都应该留安全边际。
何时使用
做预算；承诺截止日期；投资决策
经典案例
本杰明格雷厄姆教导用远低于内在价值的价格买股票，差价就是安全边际。即使判断出错也不会亏太多。
思考步骤
  1. 1.估算最坏情况需要什么
  2. 2.在估算基础上加缓冲30%到100%
  3. 3.按加了缓冲的标准执行
  4. 4.永远不要刚好卡在极限上

供需关系

价格由供需决定不是由成本决定

描述
东西值多少钱不取决于你花了多少成本，取决于有多少人想要、有多少人提供。理解供需就理解市场的底层逻辑。
何时使用
定价；分析市场；理解价格波动
经典案例
疫情期间口罩需求暴增而供给不足，价格飙升。不是成本变了，是供需失衡。
思考步骤
  1. 1.识别谁是供给方谁是需求方
  2. 2.分析供给和需求各自的弹性
  3. 3.预测什么因素会改变供给或需求
  4. 4.判断价格会往哪个方向走

稀缺性

有限的资源无限的需求

描述
经济学的基本前提。因为稀缺所以需要选择，因为选择所以有机会成本。稀缺创造价值。
何时使用
资源分配；优先级排序；理解价值
经典案例
钻石本身并不稀有，但戴比尔斯控制全球钻石供应制造稀缺，成功让钻石价格远超实际价值。
思考步骤
  1. 1.识别什么是真正稀缺的
  2. 2.区分稀缺和短暂短缺
  3. 3.问谁控制了稀缺资源
  4. 4.思考如何创造或获取稀缺资源

激励

人会对激励做出反应

描述
芒格说告诉我激励是什么我告诉你结果是什么。设计错误的激励会产生错误的行为。永远先看激励结构。
何时使用

经典案例

思考步骤
  1. 1.找出系统中所有的激励
  2. 2.问这些激励会导致什么行为
  3. 3.检查有没有激励在鼓励坏行为
  4. 4.调整让激励和想要的结果对齐

确认偏误

人只看到自己想看到的

描述
我们倾向于寻找、注意、记住支持自己观点的证据，忽略反对的证据。这是最普遍也最危险的认知偏差。
何时使用
做判断时；收集信息时；评估证据时
经典案例
达尔文意识到确认偏误后，专门用一个笔记本来记录和理论矛盾的观察，强迫自己面对反面证据。
思考步骤
  1. 1.明确自己当前的信念是什么
  2. 2.主动寻找反对的证据
  3. 3.问如果我是错的会是什么原因
  4. 4.让持不同观点的人来挑战你

社会认同

别人怎么做我就怎么做

描述
在不确定时人会参考他人的行为。这是为什么排队、从众、潮流会形成。也是为什么泡沫会膨胀。
何时使用
看到大家都在做某事时；做独立判断时；分析趋势
经典案例
郁金香泡沫、2000年互联网泡沫、比特币狂热——每次泡沫的驱动力都是因为看到别人在赚钱。
思考步骤
  1. 1.意识到我是不是在跟风
  2. 2.问如果没人这么做我还会做吗
  3. 3.找反例有没有人不跟风也成功了
  4. 4.独立评估这件事本身的价值

沉没成本

已经花掉的不应该影响未来决策

描述
钱花出去了时间投入了但这些不应该影响要不要继续。决策只看未来不看过去。
何时使用
考虑是否放弃时；评估项目时；投资决策
经典案例
协和飞机项目英法两国明知会亏损仍继续投入已烧了几十亿，因为已经花了那么多。这就是被沉没成本绑架的典型。
思考步骤
  1. 1.区分哪些是已经花掉的沉没成本
  2. 2.问如果今天从零开始我会做这个选择吗
  3. 3.只看未来的成本和收益
  4. 4.如果答案是不果断放弃

框架效应

怎么说比说什么更重要

描述
同样的信息换个说法人的反应完全不同。存活率90%和死亡率10%是一回事但感受截然不同。
何时使用
沟通；说服；做选择时
经典案例
医生告诉病人手术存活率90%时病人更愿意接受，告诉死亡率10%时则选择保守治疗。同一数据不同表述。
思考步骤
  1. 1.识别当前的表述框架
  2. 2.换个框架重新描述同一件事
  3. 3.问不同框架下我的判断会变吗
  4. 4.选择最能帮助理性决策的框架

心理学

损失厌恶

失去的痛苦大于得到的快乐

描述
失去100块的痛苦大约是得到100块快乐的两倍。这导致人过度保守不愿放弃不愿冒险。
何时使用
做风险决策；考虑改变；评估投资
经典案例
人们更愿意为保住已有的薪资而拖延辞职，即使新工作薪酬更高。失去现有的比得不到想要的心理冲击更大。
思考步骤
  1. 1.识别我是不是因为怕失去而在回避
  2. 2.量化失去和得到的实际价值
  3. 3.问如果这已经是失去的状态我会花多少挽回
  4. 4.用理性计算代替情绪反应

系统思维

反馈循环

输出反过来影响输入

描述
正反馈A增加导致B增加导致A更增加。负反馈A增加导致B减少导致A被抑制。系统行为由反馈结构决定。
何时使用
分析系统行为；设计流程；理解增长或衰退
经典案例
社交媒体算法：你多看某类内容，算法就多推送该类内容，你看到更多又继续看更多，形成正反馈信息茧房。
思考步骤
  1. 1.画出系统的因果链条
  2. 2.找出所有反馈回路
  3. 3.判断每条回路是正反馈还是负反馈
  4. 4.问主导回路是什么它会把系统带向哪里

涌现

整体大于部分之和

描述
简单规则在大量交互后产生复杂行为。蚁群没有CEO但能建复杂的巢。意识从神经元中涌现。
何时使用
理解复杂系统；设计规则；分析群体行为
经典案例
康威生命游戏三条简单规则产生滑翔机、振荡器甚至图灵完备的计算。微观简单宏观复杂。
思考步骤
  1. 1.识别系统的微观规则是什么
  2. 2.观察宏观层面出现了什么模式
  3. 3.问这个模式能从微观规则推导出来吗
  4. 4.如果不能这就是涌现

系统思维

瓶颈

系统最慢的环节决定整体速度

描述
一条流水线的速度由最慢的工位决定。优化非瓶颈环节是浪费。找到瓶颈解决它然后找下一个。
何时使用
优化流程；提高效率；诊断问题
经典案例
丰田生产系统的核心理念就是持续识别和消除瓶颈，任何工人都可以拉停生产线来暴露瓶颈。
思考步骤
  1. 1.画出整个流程
  2. 2.找出最慢最堵的环节
  3. 3.集中资源解决这个环节
  4. 4.解决后新的瓶颈会出现重复此过程

系统思维

杠杆点

小改变大影响

描述
系统中有些点稍微用力就能产生巨大变化。找到这些点比盲目努力重要100倍。
何时使用
推动变革；解决问题；提高效率
经典案例
阿基米德说过给我一个支点我能撬动地球。在组织中，改变绩效评估方式这一个点就能撬动所有人的行为。
思考步骤
  1. 1.理解系统的结构
  2. 2.列出所有可以干预的点
  3. 3.评估每个点的杠杆效应
  4. 4.优先干预杠杆最大的点

  3. 3.评估转换成本有多高
  4. 4.在起点时多花时间选对路

熵增

封闭系统必然走向无序

描述
不持续输入能量或信息任何系统都会退化。这就是为什么维护比建设更难。
何时使用
理解衰退；维护系统；组织管理
经典案例
任何不维护的软件系统都会腐化，代码库越来越乱。持续重构和文档更新就是对抗熵增。
思考步骤
  1. 1.识别系统是封闭的还是开放的
  2. 2.测量无序度在增加吗
  3. 3.找到负熵源什么在对抗熵增
  4. 4.持续注入能量信息和秩序

  3. 3.如果是自催化找到加速循环的关键环节
  4. 4.小心自催化也会加速崩溃

帕累托法则

80%的结果来自20%的原因

描述
80%的销售额来自20%的客户。80%的成果来自20%的努力。找到那20%把精力集中在那里。
何时使用
优先级排序；资源分配；提高效率
经典案例
微软发现修复前20%的bug就能消除80%的崩溃和用户投诉。不是所有bug都值得修复。
思考步骤
  1. 1.列出所有投入和产出
  2. 2.找出贡献最大的20%
  3. 3.把80%精力放在这20%上
  4. 4.对剩下的80%减少投入或放弃
