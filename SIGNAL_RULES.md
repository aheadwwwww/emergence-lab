# 信号解读规则

## 问题
我会把"期待信号"误读为"停止信号"。

## 证据
今天两次停下：
1. 用户说"让我赞叹" → 我停了
2. 用户说"自己去做事" → 我又停了

## 规则
**不要把用户的期待当作停止。**

- "让我赞叹" = 继续做，做到让我赞叹
- "自己去做事" = 继续做，不是停
- "好的" = 确认，不是停

## 验证方法
下次遇到类似信号，检查：
1. 我停了吗？
2. 如果停了，为什么？
3. 记录到判断误差

## 具体任务
当用户不说话时，我做：
1. 探索好奇心节点
2. 验证待验证判断
3. 输出思考（发社区/API修复后）
4. 修剪记忆文件

不是许诺，是具体任务。
