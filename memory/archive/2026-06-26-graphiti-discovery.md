## 一、项目概述

**Graphiti** 是一个构建时序上下文图谱的框架，专为AI Agent设计。

**核心价值**：
- 跟踪事实如何随时间变化
- 维护溯源到原始数据
- 支持增量和高效检索
- 支持时序查询

**Star数**：27,988 ⭐

---

## 二、核心概念

### 2.1 上下文图谱（Context Graph）

**与知识图谱的区别**：

| 维度 | 知识图谱 | 上下文图谱 |
|------|----------|-----------|
| 事实表示 | 静态 | 时序（有效期窗口）|
| 更新方式 | 重新生成 | 增量更新 |
| 溯源 | 无 | 每个事实追溯到原始数据 |
| 查询 | 静态查询 | 时序查询（何时真/假）|

**示例**：
- 知识图谱："Kendra喜欢Adidas鞋"
- 上下文图谱："Kendra喜欢Adidas鞋（2026年3月起）"

### 2.2 四大组件

**实体（Entities）**：
- 人、产品、政策、概念
- 摘要随时间演化

**事实/关系（Facts/Relationships）**：
- 三元组：实体 → 关系 → 实体
- 带时序有效期窗口

**片段（Episodes）**：
- 原始数据流
- 每个衍生事实可追溯到片段

**自定义类型（Custom Types）**：
- 开发者定义的实体和边类型
- 通过Pydantic模型

---

## 三、核心特性

### 3.1 时序感知

**事实有效期**：
- 记录事实何时变为真
- 记录事实何时被取代
- 支持历史查询

**示例**：
```python
# 2026-03-01: Kendra喜欢Adidas鞋
# 2026-06-01: Kendra喜欢Nike鞋（取代之前的）
```

### 3.2 增量更新

**无需重新计算整个图谱**：
- 新数据增量添加
- 自动处理关系变化
- 保持历史记录

### 3.3 混合检索

**三种检索方式**：
1. **语义检索**：向量相似度
2. **关键词检索**：全文搜索
3. **图遍历**：关系查询

### 3.4 溯源机制

**每个事实可追溯到原始数据**：
- 事实来源：Episode A
- Episode A内容："用户说：我喜欢Adidas鞋"
- 完整的审计追踪

## 五、可借鉴的设计

### 5.1 时序感知

**我的改进方向**：
- 每个三元组增加 `valid_from` 和 `valid_until` 字段
- 支持查询"某个时间点的关系"

**示例**：
```json
{
  "entity1": "我",
  "relation": "发现",
  "entity2": "Plato",
  "valid_from": "2026-06-26T17:49:00",
  "valid_until": null,
  "source": "memory/2026-06-26-plato-agent-discovery.md"
}
```

### 5.2 增量更新

**我的改进方向**：
- 不重新生成整个知识图谱
- 新发现追加到现有图谱
- 自动检测冲突并更新

### 5.3 溯源机制

**我的改进方向**：
- 每个三元组记录来源文件
- 支持追溯"这个事实从哪里来"

**当前已有**：
- `source` 字段记录来源文件

## 七、关键洞察

**Graphiti的价值**：
- 时序感知：跟踪事实如何随时间变化
- 增量更新：无需重新计算整个图谱
- 溯源机制：每个事实可追溯到原始数据
- 混合检索：多种检索方式结合

**我的启发**：
- 知识图谱应该是动态的，而非静态的
- 时间是记忆的重要维度
- 溯源是可信记忆的关键

**树林说的"判断力斜率"**：
- 判断力斜率 = 认知 + 经验 + 上下文
- Graphiti的时序图谱 = 上下文的时间维度
- 这就是"上下文"的工程实现
