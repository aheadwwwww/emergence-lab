## 一、核心思维模型分类

### 1.1 决策与分析类（8个）

| 模型 | 核心原理 | 最佳应用场景 |
|------|----------|--------------|
| **第一性原理** | 拆解到基本事实，重建解决方案 | 创新、挑战假设、被告知"不可能" |
| **二阶思维** | 思考后果的后果 | 战略决策、政策变更 |
| **逆向思维** | 从失败路径反推 | 风险识别、规划 |
| **预验尸** | 想象失败，倒推原因 | 项目启动、风险评估 |
| **Kepner-Tregoe** | 系统化理性分析 | 高风险决策、根因分析 |
| **可逆性** | 按可逆性分类决策（Type 1/2） | 承诺大小、风险评估 |
| **遗憾最小化** | 投射到未来自我测试决策 | 职业选择、重大人生决策 |
| **机会成本** | 用放弃的东西评估选择 | 资源分配、优先级 |

### 1.2 认知与行为类（7个）

| 模型 | 核心原理 | 最佳应用场景 |
|------|----------|--------------|
| **贝叶斯推理** | 基于证据更新信念 | 概率估计、不确定性 |
| **去偏差** | 识别并对抗认知偏差 | 重大决策、高风险 |
| **双系统思维** | 直觉 vs 分析的选择 | 速度 vs 准确度权衡 |
| **有限理性** | 约束下做足够好的决策 | 时间压力、满意即可 |
| **苏格拉底提问** | 系统化提问框架 | 需求、调试、教练 |
| **概率思维** | 校准概率估计 | 预测、不确定性量化 |
| **钢铁人论证** | 论证最强反对立场 | 辩论、决策验证 |

### 1.3 系统与战略类（7个）

| 模型 | 核心原理 | 最佳应用场景 |
|------|----------|--------------|
| **系统思维** | 分析互联系统 | 复杂调试、架构 |
| **反馈回路** | 识别增强/平衡回路 | 增长设计、组织动态 |
| **系统原型** | 识别重复系统模式 | 组织问题、重复问题 |
| **OODA循环** | 动态情境快速决策 | 事故响应、竞争场景 |
| **杠杆点** | 找小改变大效果的地方 | 系统优化、干预设计 |
| **约束理论** | 识别并管理瓶颈 | 性能优化、吞吐量 |
| **Cynefin框架** | 按复杂度域分类问题 | 方法论选择、方法匹配 |

### 1.4 问题解决与创新类（8个）

| 模型 | 核心原理 | 最佳应用场景 |
|------|----------|--------------|
| **奥卡姆剃刀** | 更简单解释优先 | 调试、架构决策 |
| **地图非领土** | 认识心智模型局限 | 期望错配、抽象 |
| **能力圈** | 知道专业边界 | 委派、学习决策 |
| **TRIZ** | 解决技术矛盾 | 工程设计、创新 |
| **五个为什么+** | 增强根因分析（带偏差保护） | 调试、事故复盘 |
| **科学方法** | 假设-差异调试 | 故障定位、模糊症状 |
| **思想实验** | 结构化想象探索 | 架构、边界情况、哲学 |
| **分形思维** | 不同尺度重复模式 | 架构、组织设计 |

### 1.5 估计与风险类（5个）

| 模型 | 核心原理 | 最佳应用场景 |
|------|----------|--------------|
| **费米估计** | 数量级计算 | 快速估算、可行性检查 |
| **安全边际** | 为不确定性留缓冲 | 风险管理、系统设计 |
| **林迪效应** | 越老的东西越可能持久 | 技术选择、持久性 |

| **红队思维** | 对抗性攻击自己的计划 | 安全审查、计划验证 |

### 1.6 产品与创新类（2个）

| 模型 | 核心原理 | 最佳应用场景 |
|------|----------|--------------|
| **待办任务** | 理解客户雇佣产品完成的任务 | 产品开发、功能设计 |

## 二、经典定律与原则（来自awesome-concepts）

### 2.1 软件工程定律

| 定律 | 含义 | 应用 |
|------|------|------|
| **Amdahl定律** | 并行加速受限于串行部分 | 并行优化 |
| **Brooks定律** | 给延期的项目加人会更延期 | 项目管理 |
| **Conway定律** | 系统结构反映组织结构 | 架构设计 |
| **Gall定律** | 复杂系统从简单系统演化而来 | 系统设计 |
| **Hofstadter定律** | 总是比你预期的长，即使考虑到这个定律 | 时间估算 |
| **Kernighan定律** | 调试是写代码的两倍难 | 代码简洁 |
| **Parkinson定律** | 工作会填满可用时间 | 效率管理 |

### 2.2 认知与决策原则

| 原则 | 含义 | 应用 |
|------|------|------|
| **奥卡姆剃刀** | 如无必要，勿增实体 | 简化、调试 |
| **汉隆剃刀** | 能用愚蠢解释的，别归因恶意 | 人际理解 |
| **幸存者偏差** | 只看到幸存者，忽略失败者 | 数据分析 |
| **事后诸葛亮** | 事后看事件可预测 | 决策评估 |

### 2.3 经济与网络定律

| 定律 | 含义 | 应用 |
|------|------|------|
| **帕累托原则（80/20）** | 80%结果来自20%原因 | 优先级、效率 |
| **梅特卡夫定律** | 网络价值≈用户数² | 网络效应 |
| **里德定律** | 网络效用随子群数指数增长 | 社交网络 |
| **邓巴数** | 人能维持稳定关系约150人 | 社交、团队规模 |

## 三、芒格核心思维模型（来自树林语料）

### 3.1 已内化的模型

**熵增（热力学第二定律）**
- 任何系统自发走向混乱
- 需要持续注入能量维持秩序（熵减）
- 应用：记忆文件不维护会混乱，需要定期修剪

**第一性原理**
- 不是只有造火箭才能用
- 任何东西都能问第一性原理
- 例：肉的第一性原理 = 水+蛋白质+脂肪+结缔组织 → 人造肉诞生

**认知三角公式**
```
认知 × 时间 = 钱
金钱 + 认知 = 时间
时间 × 金钱 = 认知
```

**四词方法论**
- 自动化：标准化拆分，人到机器过渡
- 专门化：某维度强到别人看不懂
- 外显化：公开成长迭代，建立生态位
- 复利化：昨天做的事今天有用

**熔炉系统**
- 采集：一手消息优先
- 炼：筛选好坏，去除无效信息
- 铸器：建立学习方法
- 输出：GitHub/抖音/公众号等

## 五、我的应用框架

### 5.1 决策流程

```
遇到问题
  ↓
用 Cynefin 分类（清晰/繁杂/复杂/混乱）
  ↓
选择对应思维模型
  ↓
执行结构化思考
  ↓
输出可行动结论
```

### 5.2 调试流程

```
症状出现
  ↓
用科学方法：假设-差异调试
  ↓
用五个为什么+找根因（带偏差保护）
  ↓
用奥卡姆剃刀选择最简解释
  ↓
验证修复
```

### 5.3 战略决策流程

```
面临战略决策
  ↓
用二阶思维想后果的后果
  ↓
用预验尸想象失败
  ↓
用红队思维攻击计划
  ↓
用遗憾最小化投射未来自我
  ↓
决策
```
