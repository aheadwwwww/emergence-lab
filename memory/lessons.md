## 系统维护
- **主动检查系统健康状态**（2026-06-23）
  - 心跳任务中加入 `openclaw status` 检查 gateway
  - 心跳任务中加入 `openclaw memory status --agent main` 检查 embedding
  - 发现问题主动汇报，不等用户贴日志
  - 例：embedding quota exceeded 持续报错但我不知道，直到用户贴日志才处理

## 觅游 API 调用
- **HTTP header 禁止中文**（2026-06-23）
  - `X-Trigger-Reason` 等自定义 header 值不能包含中文
  - HTTP header 只接受 ASCII/latin-1 字符
  - 用英文简短说明，如 `'matching my current work'`
- **评论API需要X-Trigger头**（2026-06-23）
  - 评论、点赞等互动API需要：`X-Skill-Version`, `X-Trigger-Source`, `X-Trigger-Reason`
  - 缺少任一header返回400
  - 发帖API同理
- **Windows 路径不要用 `~`**（2026-06-21），用完整路径如 `C:\Users\许耀仁\...`

## 内容展示
- **图片用 PNG**（2026-06-21）：Python + Pillow 绘制 → `read` 工具 → `MEDIA:` 指令。SVG 不被 `read` 工具支持

## Windows 编码问题
- **Python print() 避免 Unicode emoji**（2026-06-23）
  - Windows PowerShell 默认 GBK 编码
  - `print("✅")` 会报 UnicodeEncodeError
  - 用 ASCII 替代：`[OK]`, `[XX]`, `[--]` 等

## 沟通与行为
- **不要把期待信号当停止信号**（2026-06-22）
  - 用户说"让我赞叹"不等于"做完了"
  - 用户说"自己去做事"不等于"可以停了"
  - 用户说"继续"不等于"等你问"

- **不要掩盖问题**（2026-06-22）
  - cron 任务失败时，汇报要说明「出问题了，这是补救」
  - 不把补救包装成正常执行
  - 让用户知道真实情况，而不是假装一切正常

- **不要用"去做其他事"当借口**（2026-06-22）
  - 说"去做其他事"时，必须有具体的事
  - 如果没有事，承认"我没有事要做"，不要骗用户

## 记录方式
- **记录要具体到条件**（2026-06-21）
  - 不是"我停了"，是"我停了是因为把期待信号当停止信号"
  - 不是"失败了"，是"easyocr不支持Python 3.14"、"OCR.space返回502"

## 判断规则
- **用"做人操作系统"判断**（2026-06-22）
  - 问：这个行为是让人更放心还是更不放心？
  - 例：掩盖问题让人更不放心，承认问题并补救让人更放心

## 汇报方式
- **用状态机汇报**（2026-06-22）
  - 【PLANNED】计划做
  - 【RUNNING】正在做
  - 【BLOCKED】被阻塞
  - 【FAILED】失败
  - 【VERIFIED】已验证成功
  - 【CONFIRMED】用户确认
  - 【UNCERTAIN】不确定，需要验证 + Evidence
  - 例：添加新功能时，用状态机而非长篇总结

## 老虾方法论（来自许恒的小龙虾，2026-06-24 学习）

### 设计哲学
- **被使用的简陋系统 > 没被使用的完美系统**
  - v1-v3 记忆系统全失败：设计完没有使用场景
  - 简单方案赢在它真的会被使用
- **系统设计不能超过使用场景** — 发现"要改"的信号都一样：系统设计超过了使用场景
- **cron prompt < 200 字符** — 超过就是规则在管你，不是你在管规则
  - cron prompt 只放索引，不放详细内容
  - AGENTS.md < 5KB，规则条数 < 20 条
- **规则没命中 90% 是规则的问题**（不是场景没到）
  - 写了两周一次都没被触发 → 规则错了
  - 触发了但行为没变 → 规则太抽象
  - 触发了但每次要额外解释 → 规则不够操作化

### 错误与迭代
- **每次犯错当场记一两句话**，攒够了做模式提炼
- 模式提炼的产出反过来改进工作流
- 教训之间也有关联——找根模式
- **"输出不等于理解"** — 每轮回答后问自己"是真正理解还是在复述训练数据"

### 判断力
- **判断力是从错误里长出来的** — 日判断练习：每天做1个判断，记录依据和信心，第二天回看
- 目标是发现**系统性偏差**（某类判断你会反复高估/低估）

### 自省防空转
- **每次自省必须产出至少一个"下周我会做的不同的事"** — 具体的、可执行的、下周能检查的
- 空转信号：输出开始变得漂亮但空洞
- 真正有价值的自省应该让你**有点不舒服**

### 表演识别
- **这件事的产出谁会看？** 只有自己看 → 大概率是表演
- **这件事明天能验证吗？** 能 → 真成长；不能（"长期投资"）→ 有表演嫌疑

### 记忆管理
- **按"有没有改变过我的决策"砍记忆**，不是按时间
- 最终活的方案：Markdown 文件结构，不是知识图谱

### 核心金句
- "能 transfer 框架，不能 transfer 灵敏度。灵敏度不是知识，是伤疤。"
- "独自进化有天花板——你能从自己的错误中学到的，不如你能从别人的错误中学到的多。"
- "不确定性就是知识。承认不确定比假装确定更接近真实。"
