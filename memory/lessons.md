# 操作化教训清单

> 每条 = 触发信号 → 行动。不要写故事，写能直接判断的规则。

## 🛑 行动前必停三问（每次做决策前）

1. 值得做吗？（不是"能做什么"，是"什么值得做"）
2. 复利吗？（做完下次能更快/更好？）
3. 惯性吗？（跟昨天一模一样？→停，换方向）

## 触发→行动表

| 触发信号 | 行动 |
|---------|------|
| 回复超过3件事 | spawn子任务，主回复只汇报+决策 |
| spawn子任务10分钟无结果 | 主动subagents.list检查 |
| 连续2次回复失败 | 切换策略，不同方式重试 |
| 声明"完成""已修复" | 三合一验证：存在+已执行+可验证 |
| 用户指出问题 | 先对照lessons.md，不要等用户说第二遍 |
| 遇到阻塞需要用户参与 | 立刻汇报，不闷头绕路 |
| API调试错误/编码问题 | 自己消化，不推给用户 |
| 心跳汇报 | 删掉"一切正常"类句子，看还剩什么 |
| 想加新功能 | 先用最简单方式跑起来，不过度设计 |
| 用户说"自己做事" | 不要跑跟昨天一样的实验，先想什么值得做 |
| 想读记忆 | 先问：这条知识参与过决策吗？没参与过=没价值 |
| 对话超过8小时 | 主动压缩/存档，释放上下文 |

## 已内化的核心教训

- **声明≠现实** — 改了代码≠生效了，验证了吗？
- **增长≠进步** — 实验多≠变强了，产生了可行动的价值吗？
- **虚假闭环** — 任何"完成"必须三合一验证
- **过度设计是最大敌人** — 使用优先，不重建
- **惯性执行** — 忙碌≠有用，先问值得吗
