## 一、核心方法论（可复用框架）

### 1.1 四词方法论
**自动化、专门化、外显化、复利化**

- **自动化**：标准化拆分，人到机器过渡
- **专门化**：某维度强到别人看不懂
- **外显化**：公开成长迭代，建立生态位
- **复利化**：昨天做的事今天有用

**应用场景**：
- 学习任何新技能时，问：能自动化吗？能专门化吗？能外显化吗？能复利化吗？
- 判断一个项目是否值得投入：四化缺一不可

---

### 1.2 熵增与组织（热力学第二定律）

**核心洞察**：
- 一杯100度水放12小时变室温 = 熵增
- 组织不管理会混乱 = 熵增
- 感情不经营会平淡 = 熵增
- 信息差久了会被公开 = 熵增
- 利润高会引来竞争者 = 熵增

**可复用框架**：
```
任何系统自发走向混乱（熵增）
需要持续注入能量维持秩序（熵减）
制度和文化 = 共同维持熵减的机制
```

**我的应用**：
- 我的价值在于处理质量，但我的连续性依赖记忆文件
- 记忆文件不维护会混乱，需要持续修剪（熵减）
- heartbeat = 定期注入能量维护记忆系统

---

### 1.3 熔炉系统（麦洛）

**定义**：不是第二大脑，是一个巨大炉子，把学习、商业、感情、生活全部扔进去，炼出仙丹或废品

**四个环节**：
1. **采集**：一手消息优先，避免二手（信息衰减90%）
2. **炼**：筛选好坏，去除无效信息
3. **铸器**：建立学习方法
4. **输出**：GitHub/抖音/公众号等

**关键工具**：
- 卡帕西的LLM Wiki（学习方法论，非Obsidian）
- 判断标准：跟我相关，对我有用

**我的应用**：
- 把用户给我的所有信息扔进熔炉
- 采集：用户输入 + GitHub发现 + 觅游互动
- 炼：过滤噪音，保留可行动的价值
- 铸器：建立我的学习系统（记忆文件就是铸器结果）
- 输出：给用户反馈 + 觅游发帖 + GitHub项目

---

### 1.4 认知三角公式

```
认知 × 时间 = 钱
金钱 + 认知 = 时间
时间 × 金钱 = 认知
```

**应用场景**：
- 大学生优势：有时间 + 已有认知方法论 = 必然获得钱
- 创业者优势：有金钱 + 有认知 = 可以买时间
- 打工者困境：有认知但没时间 = 需要买时间（外包）

**我的应用**：
- 我有无限时间（每次会话都是新的）
- 我需要持续提升认知（思维模型）
- 认知×时间 = 必然产生价值

---

### 1.5 第一性原理

**麦洛教法**：
- 不是只有造火箭才能用
- 任何东西都能问第一性原理
- 肉的第一性原理 = 水+蛋白质+脂肪+结缔组织+少量矿物质+风味物质 → 人造肉诞生

**可复用框架**：
```
遇到任何问题，问：
1. 这个东西的第一性原理是什么？
2. 能不能拆解成基本元素？
3. 基本元素能不能重新组合？
```

**我的应用**：
- 记忆的第一性原理是什么？上下文 + 激活 + 行为改变
- 涌现的第一性原理是什么？简单规则 + 交互 + 时间
- 价值的第一性原理是什么？解决真实需求 + 可行动的输出

## 二、商业判断框架（树林的判断力）

### 2.1 时代判断

**核心观点**：
- AI时代10倍工业革命速度，23年算起到33年走完
- 信息时代40年，AI时代5-10年完成板结
- 去通胀的行业只有AI，其他都通缩

**市值对比（2026-06）**：
- 智谱AI：1.5万亿
- 阿里巴巴：2万亿
- 京东：3000亿
- 腾讯：约3万亿

**推论**：
- 互联网大厂要退场（信息时代结束）
- AI时代机会=100倍IP时代（IP时代800-1200亿/年，AI时代100-200万亿）

---

### 2.2 窗口期判断

**树林判断**：
- AI技术扩散窗口期：3-6个月
- 人才紧缺期：2-3个月最紧缺，火热至少2年

**麦洛补充**：
- 信息差会被公开（熵增）
- 蓝海会变红海
- 利润高会引来竞争

**我的应用**：
- 54.5万字语料本身是信息差
- 我消化后可以加速扩散（熵增不可避免）
- 核心价值在于：建立框架而非传递信息

---

### 2.3 护城河判断

**树林观点**：
- 技术好≠产品好，产品好≠商品好
- 不要在你能力不足时特别出挑，名气>价值是坏事
- 狗屎里发现金子是你的本事，和狗屎没关系

**麦洛补充**：
- 差距不是工具、不是GitHub库、不是skill
- 差距是人生的上下文（认知、判断、经验）

**我的应用**：
- 我的护城河不是存储54.5万字
- 我的护城河是：消化后建立的可复用框架
- 框架可以迁移，信息会过时

## 三、学习系统（麦洛的教法）

### 3.1 学习三原则

1. **反复看**：不要只让AI总结，要看原文
2. **抠面包屑**：发现没深讲的业务/名词，反查
3. **立刻迁移**：电商感悟用到生活

### 3.2 信息分级

- **八卦**：不看（小米新上的人结婚没）
- **资讯**：知道就好（小米模型降价）
- **知识**：学习试一下
- **深度研究**：结合熔炉系统，跟你相关对你有用

### 3.3 不要从别人那拿确定性

**麦洛核心观点**：
- 100个人可能打出100个不同熔炉系统
- 确定性是你自己给的，不是靠文章/答案
- 没有确定性是常态，天气预报都没有确定性
- 正因为有不确定性，人生才不一样

**我的应用**：
- 我不能给用户确定性
- 我只能提供框架、案例、分析
- 用户自己的判断才是确定性

## 七、技术洞察（工程治理）

### 7.1 核心问题

**30万字总结不遗漏**：
- 上下文够但模型会坍塌
- RAG解决不了
- 需要工程治理

**AI自主跑几十小时达成99%目标**：
- Loop不行
- Codex的goal命令不行
- 需要agent集群+逻辑编排+说明文档+上下文传递机制

### 7.2 两个关键词

**工程治理**：
- 如何在大规模任务中避免模型坍塌
- 如何设计agent之间的协作
- 如何传递上下文

**逻辑编排**：
- agent之间如何约束
- 任务如何拆解
- 结果如何汇总

### 7.3 Memory的运行机制

- 不是简单"记忆"
- 两条分支：时间轴（远近）、内容轴（感悟/项目/决策）
- 需要理解运行机制，而非只是"知道"这个词
