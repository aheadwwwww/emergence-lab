## 探索：Gensim 主题建模库

### 项目简介
- **Gensim** 是一个 Python 库，专注于**主题建模**、**文档索引**和**相似度检索**
- 目标用户：NLP 和 IR（信息检索）社区
- 当前版本：4.4.0（稳定维护模式，不再接受新功能）

### 核心特性
1. **内存无关性** - 可处理超过 RAM 的大型语料库（流式、out-of-core）
2. **直观接口**
   - 流式 API：轻松接入自定义语料
   - 转换 API：轻松扩展向量空间算法
3. **高效多核实现**
   - LSA/LSI/SVD（潜在语义分析）
   - LDA（潜在狄利克雷分配）
   - RP（随机投影）
   - HDP（层次狄利克雷过程）
   - word2vec 深度学习
4. **分布式计算** - 支持集群运行 LSA 和 LDA

### 架构设计
```
gensim/
├── parsing/      # 文本解析
├── corpora/      # 语料库处理
├── matutils/     # 数学工具
├── interfaces/   # 接口定义
├── models/       # 核心模型（LDA, word2vec, Doc2Vec 等）
├── similarities/ # 相似度计算
└── utils/        # 工具函数
```

3. **参数空间探索** - 用嵌入向量分析参数空间的语义结构

# Grokking 实验结果总结

## 实验设置
- 任务：加法（mod 113）、乘法（mod 113）
- 模型：2层 Transformer，隐藏层维度 128
- 训练：5000 epochs

## 关键发现

### 顿悟现象
- **加法任务**：epoch 3000 时测试准确率从 30% 跃升到 100%
- **乘法任务**：epoch 2600 时测试准确率从 30% 跃升到 100%

### 三阶段规律
1. **记忆期**（0-1000 epoch）：训练准确率高，测试准确率低（过拟合训练数据）
2. **过拟合期**（1000-2600 epoch）：测试准确率稳定在 30% 左右
3. **顿悟期**（2600-3000 epoch）：测试准确率突然跃升到 100%

## 数学原理
- Grokking 是神经网络从**记忆**转向**泛化**的转折点
- 模型在训练过程中逐渐学会底层算法（而非死记硬背）
- 关键因素：权重衰减、数据量、模型深度
