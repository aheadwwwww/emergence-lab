# 认知工具

麦洛强调的核心：**认知是判断质量的决定因素**。

认知不是金句/信息/观点/经验/自信，是包含 **判断→决策→检查结果→反馈** 的完整系统。

## 核心工具

### 1. 第一性原理

把事物拆解到最基本组成元素再从头构建，不用类比推理。

**使用方法**：
```
问AI：[任何事物]的第一性原理是什么？
```

**案例**：
- 肉 = 蛋白质 + 水 + 脂肪 + 结缔组织 + 风味物质
- 推导出人造肉的可能性

**可问的事物**：
- AI的第一性原理
- 赚钱的第一性原理
- 学习的第一性原理
- 工作的第一性原理
- 恋爱的第一性原理

### 2. 熵增

热力学第二定律：系统无外力输入时趋向混乱。

**类比链**：
- 不管理的代码 → 乱
- 不维护的感情 → 淡
- 不修剪的MEMORY.md → 失效（700条开始崩，2000条完全失效）
- 信息差 → 公开
- 蓝海 → 红海

**对抗熵增的方法**：
- 持续输入能量（学习、思考、维护）
- 定期修剪（不只是添加）
- 建立系统（熔炉）

### 3. 芒格100个思维模型

查理·芒格提出（哈佛演讲），学会这100个可解大多数人生商业问题。

**来源**：《穷查理宝典》

**学习建议**：
- 一字一句读透
- 读复杂再读简单
- 3个月时间

### 4. 认知系统

完整的认知循环：

```
判断 → 决策 → 检查结果 → 反馈 → 更新判断
```

**关键**：
- 不是静态的知识点
- 是动态的处理过程
- 持续迭代

## 使用场景

### 今晚开始

1. 搞懂熵增
2. 搞懂第一性原理
3. 问自己"我在做的事物的第一性原理是什么"

### 学习时

- 问"这个知识的本质是什么"
- 问"这个概念的第一性原理"
- 思考"这个模型怎么迁移到其他领域"

### 决策时

- 判断：基于什么信息？一手还是二手？
- 决策：第一性原理是什么？熵增怎么影响？
- 检查：结果符合预期吗？
- 反馈：下次怎么调整？

### 维护系统时

- 我的MEMORY.md会不会熵增？
- 规则是不是越堆越多？
- 需要修剪什么？

## 与熔炉系统的关系

认知工具是熔炉系统的"燃料"：
- 第一性原理 → 帮你"炼"（筛选）
- 熵增 → 帮你判断什么需要维护
- 芒格模型 → 帮你"铸器"（建模型）

## 输出

持续输出你的认知过程：
- 分享你怎么用这些工具
- 记录你的判断和决策
- 公开你的错误和修正

输出建立影响力 → 带来更多一手信息 → 强化认知系统。

---

**记住**：认知不是终点，是过程。持续迭代，半年前的自己应该是"傻逼"。
