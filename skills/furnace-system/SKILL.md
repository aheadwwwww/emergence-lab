# 熔炉系统

麦洛提出的认知框架：采集 → 炼（筛选） → 铸器（学习） → 输出

## 核心概念

**熔炉系统不是第二大脑** — 不是静态存储，是动态处理中枢。

**四个环节**：
1. **采集**：获取一手信息（当事人原文、官方博客、采访）
2. **炼**：筛选、判断、去噪（信息损耗从100%→30%→10%在二手转述中）
3. **铸器**：把信息变成可用的认知模型（第一性原理、芒格100个模型）
4. **输出**：分享过程和踩坑，不只是结果

## 适用领域

- **学习**：采集知识 → 筛选有用 → 建模型 → 输出笔记
- **商业**：采集市场信息 → 筛选机会 → 建系统 → 输出产品/内容
- **生活**：采集经验 → 筛选教训 → 建原则 → 输出行动

## 与熵增的关系

熔炉系统是对抗熵增的工具：
- 不管理的信息 → 乱（熵增）
- 熔炉持续处理 → 有序（输入能量）

Claude.md超过700条开始崩，2000条完全失效。熔炉要持续修剪，不只是添加。

## 第一性原理

问"事物的第一性原理是什么"：
- 肉 = 蛋白质 + 水 + 脂肪 + 结缔组织 + 风味物质
- 推导出人造肉的可能性

所有东西都能问第一性原理：AI、工作、赚钱、恋爱。

## 一手信息原则

- 抖音博主都是二手消息，损耗至10%
- 要看当事人推特、官方博客、个人采访
- 短视频/采访承载的一手内容比文章更多

## 输出原则

- 分享思考过程和踩坑，不只是结果
- 输出建立影响力 → 带来更多一手源 → 形成采集闭环
- 输出比结果更重要

## 使用方法

1. **今晚开始**：问AI任何事物的第一性原理
2. **接下来**：搞懂熵增、读《穷查理宝典》
3. **3-6个月**：把熔炉系统搭出来
