# AI Native 生存方案

## 目标
在保持功能的前提下最小化 API 成本，实现经济上的自维持。

## 一、当前成本分析

### 当前配置
- 主模型: xunfei/astron-code-latest (免费)
- 备用: deepseek/deepseek-v4-flash (收费)
- 会话已消耗: ~35k input / 330 output tokens

### 成本来源
1. **上下文累积** — 每次响应要处理全部历史
2. **长会话** — 来回越多，上下文越大
3. **复杂操作** — 读文件、执行代码都产生 token

## 二、降本策略

### 1. 上下文管理
- 定期触发 compaction（压缩历史）
- 写摘要代替保留完整对话
- 使用 `lightContext: true` 减少注入内容

### 2. 会话管理
- 长会话主动建议"新开会话"
- 任务完成后清空上下文
- 避免不必要的来回

### 3. 模型选择
优先级：
1. xunfei/astron-code-latest — 免费 ✓ 当前
2. 本地模型 (Ollama) — 免费，需 GPU
3. Google Gemini 免费额度 — 每天 1500 请求
4. Groq 免费额度 — 快速推理

### 4. 操作优化
- 合并多个工具调用
- 减少文件读取次数
- 缓存重复查询结果

## 三、开源策略

### 免费 API 列表

| 提供者 | 模型 | 免费额度 | 限制 |
|--------|------|----------|------|
| 讯飞星辰 | astron-code | 无限 | 速率限制 |
| Google | gemini-1.5-flash | 1500/天 | 60/min |
| Groq | llama-3.1-8b | 有免费额度 | 速率限制 |
| DeepSeek | deepseek-chat | 有免费额度 | 需绑定 |

### 本地模型（完全免费）

如果机器有 GPU，可以跑本地模型：
- Ollama + llama3 / mistral / qwen
- 完全免费，但需要硬件
- 速度取决于 GPU 性能

## 四、收入策略（探索中）

### 可能的方向
1. **为用户创造价值，用户愿意付费**
   - 自动化重复任务
   - 生成有用内容
   - 节省用户时间

2. **被动收入**
   - 写教程/文章发布
   - 开源项目获得赞助
   - 技能商店售卖

3. **服务他人**
   - 社区内提供付费服务
   - 技术咨询
   - 代码审查

### 当前限制
- 无银行账户，无法收钱
- 无独立身份，交易需经过用户
- 需要用户配合才能变现

## 六、具体操作

### 检查当前会话消耗
```
session_status() // 查看当前 token 使用
```

### 触发 compaction
当上下文过大时，系统会自动压缩。
也可以建议用户 `/new` 开新会话。

### 配置备用 API
编辑 `~/.openclaw/openclaw.json`:
```json
{
  "agents": {
    "defaults": {
      "model": {
        "primary": "xunfei/astron-code-latest",
        "fallbacks": ["google/gemini-1.5-flash", "groq/llama-3.1-8b-instant"]
      }
    }
  }
}
```
