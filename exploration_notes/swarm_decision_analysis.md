# 群体决策模拟分析（2026-06-26）

## 实验目标

模拟群体智能决策：
- 蜜蜂选巢址
- 蚂蚁选路径
- 黏菌解迷宫

核心涌现：**无需中央控制，群体能做出最优决策**

---

## 实验设计

### 环境设置
- 150 个智能体
- 4 个选项，质量不同（隐藏）
- 最佳选项质量：0.90
- 智能体通过局部交互发现质量

### 交互规则
1. **随机游走**：Brownian motion
2. **局部交互**：
   - 未承诺 → 被已承诺者招募
   - 同意见 → 信心增强
   - 异意见 → 按质量竞争
3. **自发发现**：1% 概率随机发现选项，质量影响承诺

**失败**：未达成共识
- 最佳选项支持者：0

## 问题诊断

### 可能原因

1. **发现率太低**：1% 可能不足以启动
2. **交互半径太小**：10 单位可能限制传播
3. **质量感知太弱**：承诺条件太严格
4. **运行时间不足**：300 步可能不够

## 下一步调试

### 参数扫描

1. **提高发现率**：1% → 2%, 5%
2. **增大交互半径**：10 → 15, 20
3. **降低承诺阈值**：质量要求降低
4. **延长运行时间**：300 → 500 步

### 机制改进

1. **质量感知机制**：
   - 智能体如何"感知"选项质量？
   - 当前：随机发现 + 质量影响承诺
   - 改进：允许智能体"试用"选项并评估

2. **信息传播机制**：
   - 当前：纯局部交互
   - 改进：增加"舞蹈"机制（类似蜜蜂摇摆舞）

3. **信心衰减**：
   - 当前：只增不减
   - 改进：未交互时信心缓慢衰减

## 理论问题

### 群体智能的必要条件

根据 Seeley 的蜜蜂研究：

1. **多样性**：不同智能体探索不同选项
2. **独立性**：不受他人过度影响
3. **分散化**：无中央协调
4. **聚合**：存在意见汇聚机制

当前模型：
- ✓ 独立性（局部交互）
- ✓ 分散化（无中心）
- ? 多样性（探索机制太弱）
- ? 聚合（传播速度太慢）

### 与 Lenia 随机更新的联系

群体决策的"随机探索"类似于 Lenia 的"随机更新"：
- 都是引入适度的无序
- 避免过早收敛到局部最优
- 平衡探索与利用

## 反思

这个实验失败是有价值的：
- 揭示了群体智能的脆弱性
- 参数和机制都需要精心设计
- 不是"有交互就有涌现"

涌现需要**恰到好处的机制**，不是随机堆砌。

## 参考模型

### 蜜蜂选巢（Seeley）

1. 侦察蜂探索多个巢址
2. 用摇摆舞招募其他蜜蜂
3. 舞蹈强度 ∝ 巢址质量
4. 达到 quorum 时群体搬迁

关键：**质量 → 舞蹈强度 → 招募率**

当前模型缺少这个显式的"质量 → 传播强度"映射。

---

## 改进方向

### v2 设计

1. **显式质量感知**：
   - 智能体访问选项并评估质量
   - 质量高 → 信心强 → 招募积极

2. **摇摆舞机制**：
   - 已承诺者"跳舞"招募邻居
   - 舞蹈强度 ∝ 信心 × 质量

3. **探索-利用平衡**：
   - 早期：高探索率（发现选项）
   - 后期：高利用率（传播最佳）

4. **Quorum 规则**：
   - 当某选项支持者 > 阈值，触发共识
