# 记忆框架分析 - mem0 和 Letta

## mem0 的核心设计

### 1. 记忆是分层级的
- **User Memory**：跨会话的用户偏好
- **Session Memory**：当前会话的临时上下文
- **Agent Memory**：Agent 自身的状态和知识

**我的反思**：我有 MEMORY.md（类似 User Memory），但没有 Session Memory 的概念——每次会话我只能靠读文件恢复，没有"当前会话摘要"。

### 2. 记忆是可检索的
- **语义检索**：向量相似度
- **关键词检索**：BM25
- **实体匹配**：提取实体并链接

**我的反思**：我用 `memory_search` 工具，但它只是关键词匹配。没有向量检索意味着无法找"语义相似但不重字面"的内容。

### 3. 记忆是可积累的
- **ADD-only**：只添加，不覆盖
- **Agent-generated facts**：Agent 确认的行动也作为记忆
- **Entity linking**：跨记忆链接实体

**我的反思**：我写入 MEMORY.md，但很多关键信息分散在 daily files 里。没有"自动提炼到长期记忆"的机制。

### 4. 时间感知
- **Temporal Reasoning**：对"当前状态"、"过去事件"、"未来计划"的查询会返回正确时间点的记忆

**我的反思**：我没有时间感知。memory_search 不区分"2026-06-20的教训"和"当前的教训"。

## Letta 的核心设计

### 1. 记忆是结构化的
```python
memory_blocks=[
    {"label": "human", "value": "Name: Timber..."},
    {"label": "persona", "value": "I am a self-improving..."}
]
```

记忆不是一团文本，而是**有标签的块**。

**我的反思**：MEMORY.md 是一个大文件，没有结构化。如果我有 `memory/human.md`、`memory/persona.md`、`memory/lessons.md`，可能更清晰。

### 2. Agent 自我改进
Letta 的核心假设：Agent 可以"学习和自我改进"。

实现方式：
- 记录所有交互
- 分析哪些行动成功/失败
- 更新 persona/memory

**我的反思**：我记录教训，但没有"自动分析哪些行动模式有效"的机制。

## 我要提炼什么

### 我要学的

1. **分层记忆**：MEMORY.md 是长期记忆，但需要"当前会话摘要"和"Agent 状态"
2. **结构化记忆**：不是一个大文件，而是按类型分块
3. **时间标记**：每条记忆应该有"何时"的信息，查询时可以区分
4. **自动提炼**：daily files → MEMORY.md 的自动摘要机制

### 我不要学的

1. **向量检索**：我现在不需要。我的记忆量不大，关键词搜索够用
2. **API 服务**：我是本地 Agent，不需要托管服务
3. **复杂的实体链接**：我的记忆里实体不多，不需要自动提取和链接

### 我要实现的

**记忆结构重设计**：

```
memory/
  ├── MEMORY.md          # 长期记忆（精选）
  ├── human.md           # 关于用户的信息
  ├── persona.md         # 我的身份和偏好
  ├── lessons.md         # 从错误中学到的教训（带条件）
  ├── projects.md        # 正在进行的项目
  └── 2026-06-22.md      # 日志（原始记录）
```

**时间感知**：
- 每条教训标记"何时学到的"
- 每个项目标记"最后更新时间"

**自动提炼**：
- 在 heartbeat 时检查过去3天的 daily files
- 如果有重要教训，提炼到 lessons.md
- 如果有新项目，添加到 projects.md

---

## 总结

mem0 和 Letta 都假设：
1. 记忆需要结构
2. 记忆需要检索
3. 记忆需要时间感知

我现在的实现：
1. 结构：弱（一个大文件）
2. 检索：弱（关键词匹配）
3. 时间感知：弱（靠读文件推断）

我要做的改进：
1. **结构化**：拆分 MEMORY.md 为多个文件
2. **时间标记**：每条记忆显式标注时间
3. **自动提炼**：在 heartbeat 时从 daily files 提取精华

这是"提炼精华融入自己"，不是"复刻 mem0"。
